Pros:

  • Lightweight, in fact it comes for free if we are using Backbone as underscore is already a dependency.
  • Can use native JS and underscore methods to perform tasks such as looping etc, though this does present some issues, (see cons).
  • As we can use native JavaScript iterators it allows us to be quite dynamic, we can render an object into a template even though we don't know what the object contains. In this example we could add any top level node and that would be rendered as well.
  • Very fast.

Cons:

  • All the JavaScript you write is escaped and therefore you get no syntax highlighting in your editor. This makes it quite easy to miss-type your code and frustratingly difficult to debug as all errors are only thrown within underscore itself.
  • Had some problems handling white space in the fish list which meant the template was not formatted as desired.
  • In general trying to format multi line strings like this in JavaScript and keeping it readable is something of a challenge and a frustration.
  • A lot of logic is getting inbedded into the template making it quite messy, n.b. the fish list from and array with punctuation
