Templating comparison: the problem

This page is an example of the sort of code we usually end up with when we don't use a templating framework. View the source code.

Perhaps the code here isn't "the perfect example" of how to do this, it could probably be tidied somewhat. However, it does show how even with this relatively simple example code can rapidy become verbose and inpenetrable.

Common problems:

  • Bits of mark up scattered in amongst logic
  • Difficult to ammend or debug
  • Business logic can become deeply embedded into presentational code
  • Frequent repetition of code

Pros of using a templating library:

  • Seperation of concerns (MVC)
  • Developers can update presentational layer without having to fully understand underlying logic
  • Cleaner code
