Pros:

  • Nice template syntax, not super keen on the double braces but I can live with it.
  • Pretty fast, compiling takes most of the time, so with the correct strategy I think this could be very performant.
  • The ability to register your own helpers allows great flexibility in interpreting data though it does start to blur the lines between pure templating and programatically generating mark up in my mind.
  • Clear documentation though I had some trouble implementing the helpers as specified.

Cons:

  • Again we needed to use our revised data structure here but as mentioned with Dust, this is maybe a good thing.
  • Our comma seperated list helper was pretty much a pure JavaScript method to create a block of html and return it into the template. Okay we can dump all these methods into one place and maintain some level of abstraction but it feels a little bit at odds to use method to call a template to call a method. Perhaps this could be done better with further understainding.
  • As per Dust some implicit knowledge of the data is required though custom helpers could be used where true dynamism is required.
